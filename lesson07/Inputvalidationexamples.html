Form Input Validation Examples

We will only be working with Client-Side validation in the following examples.

Examples using built-in browser HTML validation (no javascript)

Types of input validation:

  • required - Specifies whether a field needs to be filled in before the form can be submitted.
  • minlength - Minimum length of a text input.
  • maxlength - Maximum length of a text input.
  • min - Minumum numerical value.
  • max - Maximum numerical value.
  • type - Specified the type of input. (text, number, password, etc.)
  • pattern - Specifies1 a "regular expression" that defines a pattern that the input needs to follow.

Input Element CSS pseudo-classes:

  • :valid - Allows you to apply specific styles to valid elements.
  • :invalid - Allows you to indicate with a style that an element is not valid.
  • :out-of-range - Depending on the element like a number a style can be applied.

Summary

  • Input validation can be performed with native HTML
  • More things can be done using javascript, like enabling buttons, and changing message text.
  • Input validation moves some of the workload off of the backend server and onto the client computer.
  • Input validation helps reduce incorrect or data that is not formatted corectly from getting on the the back end server.
